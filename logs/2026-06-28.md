## Decisões
- Fazer os 3 layouts (A/B/C) na mesma sessão porque a infraestrutura (`FormView`, `parseSections`, `SectionContent`) é construída uma vez e compartilhada. Adicionar B e C depois exigiria reabrir o mesmo arquivo.
- `jsonModes` separado de `structModes` porque o FormView usa `JSON.parse` — YAML/XML renderizariam erro de parse, experiência confusa.
- Sub-switcher de layout persistido em `localStorage` (`fv-json-layout`) para não perder a preferência ao trocar de arquivo.
- Array de objetos com schema uniforme → tabela automática (`isArrayOfObjects`). Array de primitivos → chips. Objeto → grid de campos.

## Aprendizados / armadilhas
- Para conteúdo com indentação significativa em ASU `replace_file`, usar block scalar `|` e escrever cada linha do arquivo com 2 espaços extras (relativos ao YAML). O YAML strips a indentação mínima, preservando o resto.
- Nomenclatura de ASUs: `AAAA-MM-DD_NNN_descricao.yaml` — acordado nesta sessão para auditabilidade.
- Entregar o YAML como arquivo para download em vez de mostrar no chat.

## Onde parei
ASU gerado e entregue; meta/ atualizado. O usuário ainda precisa aplicar o ASU com `python -m src apply`. Após aplicar, o modo Formulário estará disponível: abrir `.json` → clicar "Formulário" → sub-switcher `Cards | Tabs | Painel` aparece na toolbar.
